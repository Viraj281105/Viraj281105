### ğŸ” LLM Hallucination Detection â€” Production-Ready Verification Pipeline

<img src="https://img.shields.io/badge/Status-Complete-brightgreen?style=flat-square"/> <img src="https://img.shields.io/badge/Domain-LLM%20Reliability%20%7C%20NLI-FF6B6B?style=flat-square"/> <img src="https://img.shields.io/badge/Benchmarked-5%2B%20LLM%20Variants-orange?style=flat-square"/>

**The Problem:** LLMs hallucinate confidently. Existing detection methods are either too shallow (keyword matching) or too slow (full re-generation). There's no lightweight, calibrated, production-deployable solution.

**The Approach:**
- Formulated hallucination detection as a **conditional inference problem**: given retrieved evidence `E` and generated claim `C`, estimate `P(entailment | E, C)`
- Fine-tuned **RoBERTa-large NLI** classifiers; evaluated **probabilistic calibration** via precisionâ€“recall curves and confidence distribution analysis
- Designed independent microservices for retrieval, entailment, and confidence scoring â€” enabling **hot-swappable model backends**
- Analyzed **performance degradation under distribution shift** across retrieval configurations
- Explored **continual adaptation mechanisms** to mitigate model drift as LLMs evolve

**Architecture:**
```
LLM Response â†’ Dense Vector Retrieval (pgvector)
                        â†“
            Evidence Ranking & Context Assembly
                        â†“
         RoBERTa-large NLI Entailment Scoring
                        â†“
    Confidence Calibration â†’ XAI Verdict (Gradio UI)
```

**Results:** >25% improvement in factual reliability on benchmark datasets; benchmarked across 5+ LLM variants on factuality, precision, and calibration metrics.

`RoBERTa` `HuggingFace` `RAG` `pgvector` `FastAPI` `Gradio` `SHAP` `PostgreSQL` `Continual Learning`
